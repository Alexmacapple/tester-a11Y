# Frameworks de Storytelling et de Structure
## Guide Complet pour Présentations de Conseil

Ce document détaille les principaux frameworks utilisés dans les présentations de conseil professionnel pour structurer le contenu, argumenter et convaincre efficacement.

---

## 1. SCQA (Situation, Complication, Question, Answer)

### Description
Framework développé par Barbara Minto, privilégié dans le conseil stratégique pour structurer une argumentation logique et convaincante.

### Structure

**S - Situation** (2-3 slides)
- Poser le contexte actuel
- Décrire l'environnement
- Établir les faits incontestables
- Créer un terrain d'entente avec l'audience

**C - Complication** (2-3 slides)
- Identifier le problème ou l'obstacle
- Montrer ce qui ne fonctionne pas
- Créer la tension narrative
- Utiliser des données pour illustrer

**Q - Question** (1 slide)
- Formuler LA question clé à résoudre
- Question doit être spécifique et actionnelle
- "Comment pouvons-nous...?"
- Point de bascule de la présentation

**A - Answer** (3-5 slides)
- Fournir les recommandations
- Structurer en 3-5 points MECE
- Appuyer par des preuves
- Conclure avec les prochaines étapes

### Quand l'utiliser
- Présentations stratégiques
- Recommandations complexes
- Contexte d'entreprise en évolution
- Audience senior qui nécessite du contexte

### Exemple de structure (10 slides)
1. Titre
2. Situation : Marché actuel
3. Situation : Performance de l'entreprise
4. Complication : Perte de parts de marché
5. Complication : Analyse des causes
6. Question : Comment retrouver la croissance?
7. Answer : Stratégie recommandée
8. Answer : Plan d'action détaillé
9. Answer : Impacts attendus
10. Conclusion et prochaines étapes

## 2. Principe de la Pyramide (Minto Pyramid)

### Description
Méthode de Barbara Minto, standard dans le conseil pour structurer une argumentation de manière déductive. Message principal en premier, puis arguments de support.

### Structure

**Niveau 1 : Message Principal** (1 slide)
- Recommandation ou conclusion principale
- Réponse à la question posée
- "Bottom line" immédiatement visible

**Niveau 2 : Arguments Principaux** (3-5 slides)
- 3 à 5 arguments majeurs (idéalement 3)
- Chaque argument soutient le message principal
- Arguments organisés de manière MECE

**Niveau 3 : Preuves et Données** (variables)
- Données, faits, analyses
- Chaque preuve soutient un argument
- Niveau de détail variable selon besoin

### Principes clés
- **Logique verticale** : chaque niveau supporte le niveau supérieur
- **Logique horizontale** : éléments du même niveau sont MECE
- **Règle de 3** : idéalement 3 arguments par niveau

### Quand l'utiliser
- Présentations exécutives (time is money)
- Recommandations claires nécessaires immédiatement
- Audience qui veut la conclusion d'abord
- Comités de direction

### Exemple de structure (9 slides)
1. Titre
2. **Recommandation principale : Acquérir la société X**
3. Argument 1 : Synergies commerciales de 50M€
4. Preuve 1.1 : Analyse détaillée des synergies
5. Preuve 1.2 : Comparables de marché
6. Argument 2 : Fit culturel fort
7. Preuve 2.1 : Résultats de due diligence
8. Argument 3 : Timing optimal
9. Plan d'action et prochaines étapes

## 3. AIDA (Attention, Intérêt, Désir, Action)

### Description
Framework marketing classique, excellent pour les présentations commerciales et de persuasion. Parcours émotionnel progressif.

### Structure

**A - Attention** (1-2 slides)
- Capter l'attention immédiatement
- Fait surprenant, question provocante, statistique
- Visuels impactants
- Créer la curiosité

**I - Intérêt** (2-3 slides)
- Développer le sujet
- Montrer la pertinence pour l'audience
- Présenter les enjeux
- Maintenir l'engagement

**D - Désir** (3-4 slides)
- Présenter la solution/produit
- Montrer les bénéfices concrets
- Créer l'envie
- Témoignages, preuves sociales

**A - Action** (1-2 slides)
- Appel à l'action clair et spécifique
- Enlever les freins
- Urgence / scarcité si applicable
- Prochaines étapes concrètes

### Quand l'utiliser
- Présentations commerciales
- Lancement de produit
- Levée de fonds
- Convaincre d'adopter une solution

### Exemple de structure (12 slides)
1. Titre
2. Attention : "70% des entreprises échouent dans leur transformation digitale"
3. Intérêt : Les 3 raisons de l'échec
4. Intérêt : Votre situation actuelle
5. Désir : Notre solution unique
6. Désir : Comment ça fonctionne
7. Désir : Résultats clients (témoignages)
8. Désir : ROI attendu
9. Action : Forfaits proposés
10. Action : Prochaines étapes
11. Action : Offre spéciale si décision rapide
12. Merci et contact

## 4. PASS (Problème, Agitation, Solution, Situation)

### Description
Framework puissant pour présenter un problème et sa solution. Intensifie l'urgence avant de présenter la solution.

### Structure

**P - Problème** (2-3 slides)
- Identifier clairement le problème
- Le nommer explicitement
- Données factuelles
- Créer la reconnaissance

**A - Agitation** (2-3 slides)
- Intensifier l'urgence
- Montrer les conséquences de l'inaction
- Coûts, risques, opportunités manquées
- Créer l'inconfort

**S - Solution** (3-4 slides)
- Présenter la solution
- Comment elle résout le problème
- Caractéristiques et avantages
- Différenciation

**S - Situation** (2-3 slides)
- Décrire la situation future améliorée
- Bénéfices concrets
- Témoignages / preuves
- Vision positive

### Quand l'utiliser
- Problème clair à résoudre
- Besoin de créer l'urgence
- Solution innovante à présenter
- Changement organisationnel

### Exemple de structure (12 slides)
1. Titre
2. Problème : Votre infrastructure IT est obsolète
3. Problème : Données de performance
4. Agitation : Coût de maintenance croissant (+30%/an)
5. Agitation : Risques de cybersécurité
6. Agitation : Perte de compétitivité
7. Solution : Notre plateforme cloud intégrée
8. Solution : Architecture technique
9. Solution : Plan de migration
10. Situation : Réduction des coûts de 40%
11. Situation : Témoignage client similaire
12. Prochaines étapes

## 5. What / So What / Now What

### Description
Framework simple et efficace pour les mises à jour, comptes-rendus et communications de changement. Structure en trois temps clairs.

### Structure

**What** (3-4 slides)
- Qu'est-ce qui s'est passé?
- Faits, données, événements
- Situation actuelle
- Contexte neutre et factuel

**So What** (3-4 slides)
- Pourquoi c'est important?
- Impact sur l'organisation
- Conséquences
- Signification stratégique

**Now What** (2-3 slides)
- Que fait-on maintenant?
- Plan d'action
- Prochaines étapes
- Décisions à prendre

### Quand l'utiliser
- Compte-rendu de projet
- Communication de changement
- Update exécutif
- Post-mortem
- Présentation de résultats

### Exemple de structure (10 slides)
1. Titre : Update Q3
2. What : Résultats financiers Q3
3. What : Faits marquants opérationnels
4. What : Changements organisationnels
5. So What : Impact sur les objectifs annuels
6. So What : Risques identifiés
7. So What : Opportunités à saisir
8. Now What : Ajustements stratégiques
9. Now What : Plan d'action Q4
10. Conclusion et décisions requises

## 6. MECE (Mutually Exclusive, Collectively Exhaustive)

### Description
Principe fondamental du consulting pour structurer l'information de manière exhaustive et non-redondante.

### Principes

**Mutually Exclusive (Mutuellement Exclusif)**
- Aucun chevauchement entre catégories
- Chaque élément appartient à une seule catégorie
- Évite la confusion et la redondance

**Collectively Exhaustive (Collectivement Exhaustif)**
- Couvre tous les éléments possibles
- Rien n'est oublié
- Vision complète de la problématique

### Applications Courantes

**Segmentation 2×2**
```
        |  Faible  |  Forte  |
--------|----------|---------|
Urgent  |   III    |    I    |
--------|----------|---------|
Non-    |   IV     |   II    |
urgent  |          |         |
```

**Catégorisation 3×3**
- Produits : Nouveaux / Existants / En déclin
- Marchés : Domestique / International / Émergent
- Clients : Grands comptes / PME / TPE

**Arbres de décision**
- Niveaux hiérarchiques MECE
- Chaque branche exclusive
- Ensemble exhaustif des options

### Quand l'utiliser
- Segmentation de marché
- Analyse de portfolio
- Arbres de décision
- Catégorisation de problèmes
- Organisation d'arguments

### Conseils pratiques
- Vérifier l'exclusivité : aucun élément dans 2 catégories
- Vérifier l'exhaustivité : tous les cas couverts
- Tester avec des exemples concrets
- Catégorie "Autres" si nécessaire (temporaire)

## 7. Storytelling Classique (3 Actes)

### Description
Structure narrative traditionnelle adaptée aux présentations business. Crée un arc narratif engageant.

### Structure

**Acte 1 : Exposition** (25% des slides)
- Présentation du contexte
- Personnages (l'entreprise, le marché)
- Situation initiale
- Enjeux

**Acte 2 : Confrontation** (50% des slides)
- Obstacles et défis
- Tension narrative
- Tentatives de solutions
- Complications

**Acte 3 : Résolution** (25% des slides)
- Solution apportée
- Résultats obtenus
- Nouvelle situation
- Leçons apprises

### Quand l'utiliser
- Études de cas
- Success stories
- Présentation de transformation
- Inspiration et motivation

## 8. Tableau Comparatif des Frameworks

| Framework | Complexité | Durée idéale | Audience | Type de présentation |
|-----------|------------|--------------|----------|---------------------|
| SCQA | Moyenne | 30-45 min | Executives | Stratégie, Recommandations |
| Pyramide | Élevée | 20-30 min | C-level | Décisions, Recommandations |
| AIDA | Faible | 15-30 min | Commerciale | Vente, Marketing |
| PASS | Moyenne | 20-40 min | Mixte | Problème-Solution |
| What/So What/Now What | Faible | 15-25 min | Opérationnelle | Updates, Comptes-rendus |
| MECE | Élevée | Variable | Analytique | Structuration d'analyse |
| Storytelling | Moyenne | 20-40 min | Large | Cas clients, Inspiration |

## 9. Combiner les Frameworks

### SCQA + Pyramide
Structure globale en SCQA, chaque section (notamment Answer) structurée en pyramide.

```
SCQA niveau macro:
- Situation (slides 1-2)
- Complication (slides 3-4)
- Question (slide 5)
- Answer (slides 6-10)
  ↓
  Answer structuré en Pyramide:
  - Recommandation principale (slide 6)
  - Argument 1 + preuves (slides 7-8)
  - Argument 2 + preuves (slide 9)
  - Argument 3 (slide 10)
```

### AIDA + PASS
Utiliser AIDA pour l'ouverture, PASS pour le développement.

```
- Attention (slide 1-2) = Accroche AIDA
- Intérêt (slide 3) = Problème PASS
- Désir (slides 4-7) = Agitation + Solution PASS
- Action (slide 8-9) = Situation PASS + CTA
```

## 10. Checklist de Sélection de Framework

Répondez à ces questions pour choisir le bon framework :

1. **Quel est l'objectif ?**
   - Convaincre → AIDA
   - Informer → What/So What/Now What
   - Recommander → SCQA, Pyramide

2. **Qui est l'audience ?**
   - C-level → Pyramide
   - Commercial → AIDA
   - Opérationnel → What/So What/Now What

3. **Quel est le contexte ?**
   - Problème clair → PASS
   - Situation complexe → SCQA
   - Update régulier → What/So What/Now What

4. **Combien de temps ?**
   - < 20 min → AIDA, What/So What/Now What
   - 20-40 min → SCQA, PASS
   - > 40 min → SCQA + Pyramide combinés

5. **Quel est le niveau de détail requis ?**
   - Élevé → Pyramide, MECE
   - Moyen → SCQA, PASS
   - Faible → AIDA, What/So What/Now What

## Anti-Patterns à Éviter

### ❌ Anti-Pattern 1 : SCQA Déséquilibré

**Erreur courante** : Situation qui occupe 60% de la présentation

**Symptômes** :
- Slides 1-10 : Contexte, historique, données de marché (trop de setup)
- Slide 11 : Complication (trop tard, audience décroche)
- Slide 12-15 : Question et Answer précipitées

**Impact** : L'audience s'ennuie pendant la Situation et ne retient pas la Solution.

**Solution** :
- Situation : 20-25% des slides max
- Complication : 20-25%
- Question : 5-10% (1-2 slides)
- Answer : 40-50% (le plus important !)

**Exemple** (15 slides) :
- Situation : 3 slides
- Complication : 3 slides
- Question : 1 slide
- Answer : 7 slides
- Conclusion : 1 slide

---

### ❌ Anti-Pattern 2 : Question Vague ou Multiple

**Erreur courante** : Plusieurs questions ou question trop large

**Mauvais exemples** :
- ❌ "Que devons-nous faire ?" (trop vague)
- ❌ "Comment améliorer nos ventes ET réduire nos coûts ET innover ?" (3 questions)
- ❌ "Quelle est notre stratégie ?" (pas actionnée)

**Bons exemples** :
- ✅ "Comment gagner 5% de PDM d'ici 18 mois ?"
- ✅ "Quel produit lancer en priorité pour le Q2 ?"
- ✅ "Faut-il acquérir la société X maintenant ?"

**Règle d'or** : UNE question, SPÉCIFIQUE, ACTIONNÉE

---

### ❌ Anti-Pattern 3 : Answer Non Structurée

**Erreur courante** : Réponse en vrac sans structure MECE

**Symptômes** :
- Slide 8 : "Première idée : améliorer le pricing"
- Slide 9 : "Aussi : former les commerciaux" (lié au pricing ?)
- Slide 10 : "Et puis : nouveau packaging" (chevauchement avec pricing ?)
- Slide 11 : "Sans oublier : publicité en ligne" (nouveau pilier ou partie de quoi ?)

**Impact** : L'audience ne comprend pas la logique et retient mal.

**Solution** : Structure MECE claire

✅ **Answer structurée** :
```
Slide 7 : "3 leviers complémentaires pour gagner 5% PDM"
Slide 8 : Levier 1 - PRICING (- 15% sur entrée gamme)
Slide 9 : Levier 2 - DISTRIBUTION (50 nouveaux points)
Slide 10 : Levier 3 - MARKETING (campagne digitale 500k€)
Slide 11 : Roadmap et investissements (vue intégrée)
```

---

### ❌ Anti-Pattern 4 : Pyramide Inversée

**Erreur courante** : Commencer par les détails, finir par la conclusion

**Symptômes** :
- Slide 1-5 : Données, analyses, graphiques
- Slide 6-10 : Détails techniques
- Slide 11 : "Donc, nous recommandons..." (trop tard !)

**Impact** : C-level perd patience, part avant la fin ou ne retient pas le message.

**Solution** : Principe de la Pyramide de Minto

✅ **Structure Pyramide** :
```
Slide 1 : Recommandation principale (le "quoi")
Slide 2 : "Pourquoi cette recommandation : 3 raisons"
Slide 3-5 : Détail des 3 raisons (arguments niveau 1)
Slide 6-10 : Preuves et données (arguments niveau 2)
Slide 11 : Recap et next steps
```

**Règle** : Message principal en PREMIER, preuves ensuite.

---

### ❌ Anti-Pattern 5 : AIDA Sans Action Claire

**Erreur courante** : Finir par "Merci" au lieu d'un appel à l'action

**Symptômes** :
- Attention ✅ : Slide 1-2 (accroche)
- Intérêt ✅ : Slide 3-5 (enjeux)
- Désir ✅ : Slide 6-9 (solution)
- Action ❌ : Slide 10 "Merci, des questions ?"

**Impact** : L'audience est convaincue mais... ne sait pas quoi faire ensuite.

**Solution** : Action SPÉCIFIQUE et URGENTE

✅ **Action claire** :
```
Slide 10 : "3 décisions attendues cette semaine :"
- 1. Valider le budget 2.5M€ (Comité jeudi)
- 2. Nommer le chef de projet (proposition : Marie)
- 3. Kickoff équipe le 15 novembre

"Qui fait quoi d'ici vendredi ?"
```

---

### ❌ Anti-Pattern 6 : MECE Non Respecté

**Erreur courante** : Catégories qui se chevauchent

**Exemple de segmentation non-MECE** :

❌ **Mauvaise segmentation** :
```
Nos 3 segments clients :
1. Clients grands comptes (>1000 employés)
2. Clients internationaux (hors France)
3. Clients digitaux (e-commerce uniquement)
```

**Problème** : Un client peut être dans 2-3 catégories !
- Google France = grand compte ✓ + international ✓ + digital ✓

✅ **Bonne segmentation MECE** :
```
Nos 3 segments clients (par taille uniquement) :
1. Grands comptes (>1000 employés) : 12 clients
2. PME (50-1000 employés) : 450 clients
3. TPE (<50 employés) : 2000 clients

Total = 2462 clients ✓ Exhaustif ✓ Exclusif
```

**Règle** : Chaque élément dans UNE SEULE catégorie, TOUTES les possibilités couvertes.

## Tableau des Anti-Patterns par Framework

| Framework | Anti-Pattern Principal | Symptôme | Solution |
|-----------|----------------------|----------|----------|
| SCQA | Situation trop longue | >40% des slides en contexte | Max 25% en Situation |
| Pyramide | Message principal à la fin | Conclusion slide 15/15 | Message slide 1-2 |
| AIDA | Pas d'appel à l'action | Dernier slide = "Merci" | "3 décisions à prendre" |
| PASS | Agitation insuffisante | Pas d'urgence créée | Chiffrer les coûts de l'inaction |
| MECE | Catégories qui se chevauchent | Éléments dans plusieurs cases | Tester avec exemples concrets |
| What/So What/Now What | "So What" absent | Données brutes sans interprétation | Ajouter "Impact : ..." après chaque fait |

## Checklist Anti-Patterns

Avant de finaliser une présentation, vérifier :

- [ ] Mon message principal est-il visible slide 1-2 ? (pas slide 15)
- [ ] Ma question centrale (si SCQA) est-elle unique et spécifique ?
- [ ] Mes catégories (si MECE) sont-elles vraiment exclusives ?
- [ ] Mon appel à l'action (si AIDA/PASS) est-il concret et daté ?
- [ ] Ma structure est-elle équilibrée ? (pas 70% setup + 30% solution)
- [ ] Mes arguments (si Pyramide) sont-ils vraiment en 3-5 points max ?
- [ ] Mes slides "Answer" représentent-elles 40-50% de la présentation ?
