Testons les contrastes

Est-ce que les contrastes sont respectés sur ce site ?

Appliquer des contrastes suffisamment élevés

L'utilisation de contrastes trop faibles diminue considérablement la lisibilité des contenus.

Impact sur les utilisateurs

Dans ce cas, les personnes malvoyantes ne perçoivent pas certains textes, composants d'interface ou éléments graphiques.

Elles sont alors incapables de lire le contenu ou d'interagir avec les éléments concernés.

Le contraste : ce n'est pas juste de l'esthétique. C'est de l'accessibilité.

Exemples de contrastes non conformes

Non conforme Texte normal - Ratio requis : 4,5:1

Texte avec contraste insuffisant rendant la lecture difficile pour les personnes malvoyantes ou daltoniennes. Ce texte a un contraste de seulement 1,6:1 (très loin des 4,5:1 requis).

Non conforme Grand texte - Ratio requis : 3:1

Ce grand texte a un contraste de 1,6:1 (insuffisant pour les 3:1 requis).

Les textes de grande taille (≥ 18pt ou ≥ 14pt gras) nécessitent un ratio de 3:1 minimum.

Non conforme Éléments d'interface - Ratio requis : 3:1

Les pictogrammes, icônes informatives et composants graphiques doivent avoir un ratio minimum de 3:1 avec l'arrière-plan pour être perceptibles.

Ces pictogrammes ont un contraste d'environ 2,5:1 (insuffisant pour les 3:1 requis) et sont difficiles à percevoir.

Les ratios de contraste à respecter

Ratios de contraste requis selon les WCAG 2.1 niveau AA
Type d'élément Caractéristiques Ratio minimum
Texte normal < 18 pt ou < 14 pt gras 4,5:1
Grand texte ≥ 18 pt ou ≥ 14 pt gras 3:1
Éléments UI Composants d'interface, icônes informatives 3:1

Critères RGAA associés

Critère 3.2

Dans chaque page web, le contraste entre la couleur du texte et la couleur de son arrière-plan est-il suffisamment élevé (hors cas particuliers) ?

Critère 3.3

Dans chaque page web, les couleurs utilisées dans les composants d'interface ou les éléments graphiques porteurs d'informations sont-elles suffisamment contrastées (hors cas particuliers) ?
